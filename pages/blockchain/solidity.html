Solidity Fundamentals

How Solidity Works

Solidity is a compiled language for writing smart contracts on EVM (Ethereum Virtual Machine) blockchains.

You write .sol files → Compiler (solc) → Bytecode → Deployed on blockchain

The flow:

  1. You write Solidity code (.sol files)
  2. The compiler (solc) translates it to bytecode (machine code for the EVM)
  3. Hardhat uses solc under the hood when you run npx hardhat compile
  4. The bytecode gets deployed to a blockchain as a smart contract
  5. Users interact with the contract by sending transactions

Syntax

File Structure

Every Solidity file follows this structure:

// 1. License identifier (required)
// SPDX-License-Identifier: MIT

// 2. Pragma — tells the compiler which version to use
pragma solidity ^0.8.28;
//              ^0.8.28 means "0.8.28 or higher, but below 0.9.0"
//              The ^ is like npm versioning — compatible within minor version

// 3. Imports
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

// 4. Contract definition (like a class in other languages)
contract MyContract {
    // state variables, functions, events, etc.
}

Data Types

// === VALUE TYPES ===

bool isActive = true;                    // true or false

uint256 amount = 100;                    // unsigned integer (0 to 2^256-1)
uint8 smallNumber = 255;                 // unsigned 8-bit (0 to 255)
int256 temperature = -50;                // signed integer (-2^255 to 2^255-1)

address wallet = 0xAbC123...;            // 20-byte Ethereum address
address payable recipient = payable(wallet); // address that can receive ETH

bytes32 hash = keccak256("hello");       // fixed-size byte array
bytes data = hex"001122";               // dynamic-size byte array
string name = "Alice";                   // UTF-8 string (dynamic)


// === REFERENCE TYPES ===

uint256[] numbers;                       // dynamic array
uint256[5] fixed;                        // fixed-size array (5 elements)

mapping(address => uint256) balances;    // key-value store (like a dictionary/hashmap)
// mappings can be nested:
mapping(address => mapping(address => uint256)) allowances;

struct User {                            // custom type (like a struct in C)
    address wallet;
    uint256 balance;
    bool isActive;
}

enum Status { Pending, Active, Closed }  // enumeration

Visibility & State

contract Example {

    // === STATE VARIABLES (stored on blockchain — costs gas) ===

    uint256 public count;        // public: auto-generates a getter function
    uint256 private secret;      // private: only this contract can read
    uint256 internal shared;     // internal: this contract + contracts that inherit it
    // (there is no "external" for state variables)

    // === CONSTANTS (embedded in bytecode — no storage cost) ===

    uint256 public constant MAX = 1000;         // set at compile time, never changes
    uint256 public immutable deployTime;         // set once in constructor, never changes

    constructor() {
        deployTime = block.timestamp;
    }
}

Functions

contract Functions {

    uint256 public value;

    // === FUNCTION VISIBILITY ===

    function publicFunc() public { }       // anyone can call (external + internal)
    function externalFunc() external { }   // only callable from outside the contract
    function internalFunc() internal { }   // only this contract + children
    function privateFunc() private { }     // only this contract

    // === STATE MUTABILITY ===

    // view: reads state but doesn't modify it (free to call, no gas if called externally)
    function getValue() public view returns (uint256) {
        return value;
    }

    // pure: doesn't read or modify state (free to call)
    function add(uint256 a, uint256 b) public pure returns (uint256) {
        return a + b;
    }

    // payable: can receive ETH with the call
    function deposit() public payable {
        // msg.value contains the ETH sent
        value += msg.value;
    }

    // default (no keyword): reads AND modifies state — costs gas
    function setValue(uint256 _value) public {
        value = _value;
    }

    // === MULTIPLE RETURN VALUES ===

    function getInfo() public pure returns (uint256, bool, string memory) {
        return (42, true, "hello");
    }
}

Global Variables (Available Everywhere)

// === TRANSACTION CONTEXT ===
msg.sender      // address that called this function
msg.value       // ETH sent with the call (in wei)
msg.data        // raw calldata bytes

// === BLOCK CONTEXT ===
block.timestamp // current block's timestamp (seconds since epoch)
block.number    // current block number
block.chainid   // chain ID (1 = mainnet, 84532 = base sepolia)

// === TRANSACTION CONTEXT ===
tx.origin       // original externally-owned account that started the tx chain
//                 (DANGEROUS — don't use for auth, see AccessControl exercise)

// === UNITS ===
1 ether   == 1e18          // 1 ETH = 10^18 wei
1 gwei    == 1e9           // 1 gwei = 10^9 wei
1 wei     == 1             // smallest unit
1 days    == 86400         // seconds
1 hours   == 3600
1 minutes == 60

Modifiers (Reusable Function Guards)

contract Ownable {
    address public owner;

    constructor() {
        owner = msg.sender;
    }

    // A modifier adds checks before (or after) a function runs
    modifier onlyOwner() {
        require(msg.sender == owner, "Not the owner");
        _; // <-- this is where the function body runs
    }

    // Using the modifier — mint() will revert if caller isn't owner
    function mint() public onlyOwner {
        // only runs if the modifier's require passes
    }
}

Error Handling

contract ErrorHandling {

    // require: validate inputs and conditions (refunds remaining gas)
    function withdraw(uint256 amount) public {
        require(amount > 0, "Amount must be > 0");
        require(amount <= balance[msg.sender], "Insufficient balance");
    }

    // revert: same as require but for complex conditions
    function transfer(address to) public {
        if (to == address(0)) {
            revert("Cannot send to zero address");
        }
    }

    // assert: check for invariants that should NEVER be false (no gas refund)
    function divide(uint256 a, uint256 b) public pure returns (uint256) {
        assert(b != 0); // if this fails, something is very wrong
        return a / b;
    }

    // Custom errors (gas-efficient, Solidity 0.8.4+)
    error InsufficientBalance(uint256 requested, uint256 available);

    function send(uint256 amount) public {
        if (amount > balance) {
            revert InsufficientBalance(amount, balance);
        }
    }
}

Events (Logging)

contract Events {
    // Events are logs stored on the blockchain — cheap storage, but
    // contracts can't read them. Used for off-chain apps (frontends, indexers).

    event Transfer(address indexed from, address indexed to, uint256 value);
    //             ^indexed: allows filtering/searching by this field (max 3 indexed)

    function transfer(address to, uint256 amount) public {
        // ... transfer logic ...
        emit Transfer(msg.sender, to, amount);
    }
}

Sending ETH

contract SendingETH {

    // Three ways to send ETH:

    // 1. call (RECOMMENDED) — forwards all gas, returns success bool
    function sendViaCall(address payable _to, uint256 _amount) public {
        (bool success, ) = _to.call{value: _amount}("");
        require(success, "Send failed");
    }

    // 2. transfer — forwards 2300 gas, reverts on failure (NOT recommended)
    function sendViaTransfer(address payable _to, uint256 _amount) public {
        _to.transfer(_amount);
    }

    // 3. send — forwards 2300 gas, returns bool (NOT recommended)
    function sendViaSend(address payable _to, uint256 _amount) public {
        bool success = _to.send(_amount);
        require(success, "Send failed");
    }
}

Receiving ETH

contract ReceivingETH {

    // receive() is called when ETH is sent with no calldata
    receive() external payable {
        // e.g., someone sends ETH directly to this contract address
    }

    // fallback() is called when no function matches the calldata
    // or when ETH is sent with calldata
    fallback() external payable {
        // catch-all
    }

    // Check contract's ETH balance
    function getBalance() public view returns (uint256) {
        return address(this).balance;
    }
}

Inheritance

// Base contract
contract Animal {
    string public name;

    constructor(string memory _name) {
        name = _name;
    }

    // virtual: allows child contracts to override this function
    function speak() public pure virtual returns (string memory) {
        return "...";
    }
}

// Child contract — inherits all of Animal's state and functions
contract Dog is Animal {
    constructor() Animal("Rex") { }

    // override: replaces the parent's implementation
    function speak() public pure override returns (string memory) {
        return "Woof!";
    }
}

// Multiple inheritance
contract GuideDog is Dog, Ownable {
    // inherits from both Dog and Ownable
}

Interfaces & Abstract Contracts

// Interface: defines function signatures without implementations
// Used to interact with other contracts
interface IERC20 {
    function transfer(address to, uint256 amount) external returns (bool);
    function balanceOf(address account) external view returns (uint256);
}

// Using an interface to call another contract
contract TokenUser {
    function checkBalance(address tokenAddress, address user) public view returns (uint256) {
        IERC20 token = IERC20(tokenAddress);
        return token.balanceOf(user);
    }
}

// Abstract: has at least one unimplemented function
abstract contract Base {
    function mustImplement() public virtual returns (uint256);
}

Data Location (memory vs storage vs calldata)

contract DataLocation {

    uint256[] public numbers; // state variable — always in STORAGE

    function example(uint256[] calldata input) external {
        //                     ^^^^^^^^
        // calldata: read-only, cheapest, only for external function params

        uint256[] memory temp = new uint256[](5);
        // memory: temporary, exists only during function execution, modifiable

        uint256[] storage ref = numbers;
        // storage: reference to the actual blockchain storage — modifications persist
        ref.push(1); // this modifies the actual `numbers` array on-chain
    }

    // Rule of thumb:
    // - Use calldata for external function inputs you won't modify
    // - Use memory for temporary variables
    // - Use storage when you need to modify state
}

Mappings Deep Dive

contract MappingExamples {

    // Simple mapping
    mapping(address => uint256) public balances;

    // Nested mapping (e.g., ERC20 allowances)
    mapping(address => mapping(address => uint256)) public allowances;

    // Mapping with struct values
    struct Player {
        uint256 level;
        uint256 score;
    }
    mapping(address => Player) public players;

    function example() public {
        // Set values
        balances[msg.sender] = 100;

        // Get values (default is 0 for missing keys)
        uint256 bal = balances[address(0)]; // returns 0, NOT an error

        // Delete
        delete balances[msg.sender]; // resets to 0

        // IMPORTANT: You CANNOT iterate over a mapping!
        // You cannot get its length or list all keys.
        // If you need iteration, maintain a separate array of keys.
    }
}

The EVM & Gas

┌─────────────────────────────────────────────────────┐
│                   TRANSACTION                        │
│  from: 0xABC...  to: 0xDEF...  value: 1 ETH        │
│  data: 0xa9059cbb000...  (encoded function call)     │
│  gasLimit: 100000  gasPrice: 20 gwei                 │
└─────────────────┬───────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────┐
│                     EVM                              │
│  Executes bytecode instruction by instruction        │
│  Each instruction costs GAS:                         │
│    ADD = 3 gas                                       │
│    SSTORE (write) = 20,000 gas (new) / 5,000 (update)│
│    SLOAD (read) = 2,100 gas                          │
│    CALL (external) = 2,600+ gas                      │
│                                                      │
│  Storage is PERMANENT — most expensive               │
│  Memory is TEMPORARY — cheaper                       │
│  Stack is for computation — cheapest                 │
└─────────────────────────────────────────────────────┘

Why gas matters for security:

  • Storage writes are expensive → incentivizes gas optimization → developers use unchecked {} → can introduce overflow bugs
  • External calls are dangerous → the called contract runs arbitrary code → reentrancy attacks
  • Block gas limit → unbounded loops can make functions permanently uncallable → DoS attacks

How This Relates to Your Hardhat Setup

hardhat.config.js          ← Configures which Solidity version & EVM to use
contracts/*.sol            ← Your Solidity source files
test/*.test.js             ← JavaScript tests that deploy & interact with contracts
npx hardhat compile        ← Runs solc on your .sol files → artifacts/
npx hardhat test           ← Compiles + spins up a local EVM + runs tests

When you run npx hardhat test:

  1. Hardhat downloads the Solidity compiler (solc 0.8.28)
  2. Compiles all .sol files to bytecode
  3. Starts a local in-memory blockchain (Hardhat Network)
  4. Your test file deploys contracts and sends transactions to this local chain
  5. Results are shown in the terminal — nothing goes to a real blockchain

The evmVersion: "cancun" setting tells the compiler to target the Cancun hard fork, which includes newer opcodes like mcopy that OpenZeppelin v5 uses.
