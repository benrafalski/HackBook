Solidity Fundamentals

How Solidity Works

Solidity is a compiled language for writing smart contracts on EVM (Ethereum Virtual Machine) blockchains.

You write .sol files → Compiler (solc) → Bytecode → Deployed on blockchain

The flow:

  1. You write Solidity code (.sol files)
  2. The compiler (solc) translates it to bytecode (machine code for the EVM)
  3. Hardhat uses solc under the hood when you run npx hardhat compile
  4. The bytecode gets deployed to a blockchain as a smart contract
  5. Users interact with the contract by sending transactions

Types, Variables & State

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;  // ^0.8.28 = ">=0.8.28 <0.9.0"
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract MyContract {
    // === Value Types ===
    bool isActive = true;
    uint256 amount = 100;                        // unsigned (0 to 2^256-1), also uint8..uint128
    int256 temperature = -50;                    // signed (-2^255 to 2^255-1)
    address wallet = 0xAbC123...;                // 20-byte Ethereum address
    address payable recipient = payable(wallet);
    bytes32 hash = keccak256("hello");           // fixed-size byte array
    string name = "Alice";                       // dynamic UTF-8 string

    // === Reference Types ===
    uint256[] numbers;                           // dynamic array
    uint256[5] fixed;                            // fixed-size array
    mapping(address => uint256) balances;        // key-value store (can't iterate!)
    mapping(address => mapping(address => uint256)) allowances; // nested
    struct User { address wallet; uint256 balance; bool isActive; }
    enum Status { Pending, Active, Closed }

    // === Visibility ===
    uint256 public count;       // auto-generates getter
    uint256 private secret;     // only this contract
    uint256 internal shared;    // this contract + children

    // === Constants (no storage cost) ===
    uint256 public constant MAX = 1000;          // compile-time
    uint256 public immutable deployTime;         // set once in constructor

    // === Data Location ===
    // storage: persistent on-chain (state vars). memory: temporary. calldata: read-only, cheapest.
    function example(uint256[] calldata input) external {
        uint256[] memory temp = new uint256[](5);
        uint256[] storage ref = numbers;         // reference to state — changes persist
    }
}

Functions & Control Flow

// Visibility:  public | external | internal | private
// Mutability:  view (reads, free) | pure (no state, free) | payable (receives ETH) | default (costs gas)

function get() public view returns (uint256) { return x; }
function add(uint a, uint b) public pure returns (uint) { return a+b; }
function deposit() public payable { value += msg.value; }
function set(uint v) public { value = v; }
function getInfo() public pure returns (uint256, bool, string memory) { return (42, true, "hello"); }

// Modifiers — reusable guards
modifier onlyOwner() {
    require(msg.sender == owner, "Not the owner");
    _;  // function body runs here
}
function mint() public onlyOwner { /* ... */ }

// Error handling
require(amount > 0, "Amount must be > 0");   // validate inputs (refunds gas)
revert("Cannot send to zero address");        // complex conditions
assert(b != 0);                               // invariant check (no gas refund)
error InsufficientBalance(uint256 requested, uint256 available); // custom error (gas-efficient)

// Events — logs on blockchain (cheap, for off-chain use, can't be read by contracts)
event Transfer(address indexed from, address indexed to, uint256 value); // indexed: filterable (max 3)
emit Transfer(msg.sender, to, amount);

ETH, Inheritance & Interfaces

// === Sending ETH — use call (recommended) ===
(bool success, ) = _to.call{value: _amount}("");
require(success, "Send failed");
// .transfer() and .send() forward only 2300 gas — avoid

// === Receiving ETH ===
receive() external payable { }   // ETH sent with no calldata
fallback() external payable { }  // no function match / ETH with calldata
address(this).balance             // contract's ETH balance

// === Inheritance ===
contract Animal {
    function speak() public pure virtual returns (string memory) { return "..."; }
}
contract Dog is Animal {
    function speak() public pure override returns (string memory) { return "Woof!"; }
}
contract GuideDog is Dog, Ownable { }  // multiple inheritance

// === Interfaces — call other contracts ===
interface IERC20 {
    function transfer(address to, uint256 amount) external returns (bool);
    function balanceOf(address account) external view returns (uint256);
}
IERC20 token = IERC20(tokenAddress);

// === Abstract — has unimplemented functions ===
abstract contract Base { function mustImplement() public virtual returns (uint256); }

Globals & EVM

msg.sender       // caller address          msg.value   // ETH sent (wei)
block.timestamp  // seconds since epoch     block.number // current block
block.chainid    // 1=mainnet, etc.         tx.origin    // original EOA (DANGEROUS for auth)

// Units
1 ether == 1e18;  1 gwei == 1e9;  1 wei == 1;
1 days == 86400;  1 hours == 3600; 1 minutes == 60;
EVM gas costs:  ADD = 3 | SLOAD (read) = 2,100 | SSTORE (write) = 5,000-20,000 | CALL = 2,600+
Storage = permanent (expensive) → Memory = temporary (cheaper) → Stack = computation (cheapest)

Why gas matters for security: storage writes → unchecked {} → overflow bugs; external calls → reentrancy; block gas limit → unbounded loops → DoS

Runtime Environments

Hardhat (JavaScript/TypeScript)

Most popular framework. JS ecosystem, great plugins (etherscan verify, gas reporter).

npm init -y ; npm i --save-dev hardhat ; npx hardhat init

contracts/*.sol          test/*.test.js          scripts/*.js          hardhat.config.js

npx hardhat compile                 → solc compiles .sol → artifacts/
npx hardhat test                    → compile + local EVM + run tests
npx hardhat test --grep "name"      → run specific test
npx hardhat node                    → start local chain (JSON-RPC)
npx hardhat run scripts/deploy.js   → deploy script

Uses ethers.js/viem + Chai assertions. Supports console.log in Solidity.
evmVersion: "cancun" for latest opcodes (OpenZeppelin v5).
// test/MyToken.test.js
const { expect } = require("chai");
const { ethers } = require("hardhat");

describe("MyToken", function () {
  let token, owner, addr1;

  beforeEach(async function () {
    [owner, addr1] = await ethers.getSigners();
    const MyToken = await ethers.getContractFactory("MyToken");
    token = await MyToken.deploy(1000);       // deploy with constructor arg
  });

  it("should transfer tokens", async function () {
    await token.transfer(addr1.address, 50);
    expect(await token.balanceOf(addr1.address)).to.equal(50);
  });

  it("should revert on insufficient balance", async function () {
    await expect(token.connect(addr1).transfer(owner.address, 1))
      .to.be.revertedWith("Insufficient balance");
  });
});
// scripts/deploy.js
const { ethers } = require("hardhat");

async function main() {
  const MyToken = await ethers.getContractFactory("MyToken");
  const token = await MyToken.deploy(1000);
  await token.waitForDeployment();
  console.log("Deployed to:", await token.getAddress());
}
main();

Foundry (Rust-based CLI)

Fastest compilation. Tests written in Solidity. Powerful fuzzing built-in.

curl -L https://foundry.paradigm.xyz | bash ; foundryup ; forge init my-project

src/*.sol          test/*.t.sol          script/*.s.sol          foundry.toml

forge build                              → compile
forge test                               → run tests
forge test --match-test "name"           → run specific test
anvil                                    → start local chain
forge script script/Deploy.s.sol         → deploy script

Cheatcodes: vm.prank(addr), vm.deal(addr, amt), vm.warp(time), vm.expectRevert()
// test/MyToken.t.sol
import "forge-std/Test.sol";
import "../src/MyToken.sol";

contract MyTokenTest is Test {
    MyToken token;
    address alice = makeAddr("alice");

    function setUp() public {
        token = new MyToken(1000);
    }

    function testTransfer() public {
        token.transfer(alice, 50);
        assertEq(token.balanceOf(alice), 50);
    }

    function testRevertInsufficientBalance() public {
        vm.prank(alice);                         // next call is from alice
        vm.expectRevert("Insufficient balance");
        token.transfer(address(this), 1);
    }

    function testFuzz_Transfer(uint256 amount) public {
        vm.assume(amount <= 1000);              // bound fuzz input
        token.transfer(alice, amount);
        assertEq(token.balanceOf(alice), amount);
    }
}
// script/Deploy.s.sol
import "forge-std/Script.sol";
import "../src/MyToken.sol";

contract DeployScript is Script {
    function run() external {
        vm.startBroadcast();                     // start signing txs
        new MyToken(1000);
        vm.stopBroadcast();
    }
}

Brownie (Python)

Python-based framework using web3.py. Good for Python-native devs. (Largely superseded by Ape.)

pip install eth-brownie ; brownie init

contracts/*.sol          tests/*.py          scripts/*.py          brownie-config.yaml

brownie compile                → compile
brownie test                   → run pytest tests
brownie test -k "name"         → run specific test
brownie console                → interactive Python REPL with deployed contracts
brownie run scripts/deploy.py  → deploy script

Uses web3.py + pytest. accounts[0] for test wallets.
# tests/test_my_token.py
from brownie import MyToken, accounts, reverts

def test_transfer():
    token = MyToken.deploy(1000, {"from": accounts[0]})
    token.transfer(accounts[1], 50, {"from": accounts[0]})
    assert token.balanceOf(accounts[1]) == 50

def test_revert_insufficient():
    token = MyToken.deploy(1000, {"from": accounts[0]})
    with reverts("Insufficient balance"):
        token.transfer(accounts[0], 1, {"from": accounts[1]})
# scripts/deploy.py
from brownie import MyToken, accounts

def main():
    token = MyToken.deploy(1000, {"from": accounts[0]})
    print(f"Deployed to: {token.address}")

Ape (Python)

Modern Python framework, spiritual successor to Brownie. Plugin-based, cleaner API.

pip install eth-ape ; ape init

contracts/*.sol          tests/*.py          scripts/*.py          ape-config.yaml

ape compile                    → compile
ape test                       → run pytest tests
ape test -k "name"             → run specific test
ape console                    → interactive Python REPL
ape run scripts/deploy.py      → deploy script

Uses ape.accounts, ape.project. Plugin system: ape-solidity, ape-hardhat, ape-alchemy.
# tests/test_my_token.py
import ape
import pytest

@pytest.fixture
def token(owner):
    return owner.deploy(ape.project.MyToken, 1000)

@pytest.fixture
def owner(accounts):
    return accounts[0]

def test_transfer(token, owner, accounts):
    token.transfer(accounts[1], 50, sender=owner)
    assert token.balanceOf(accounts[1]) == 50

def test_revert_insufficient(token, accounts):
    with ape.reverts("Insufficient balance"):
        token.transfer(accounts[0], 1, sender=accounts[1])
# scripts/deploy.py
from ape import project, accounts

def main():
    deployer = accounts.load("my-account")
    token = deployer.deploy(project.MyToken, 1000)
    print(f"Deployed to: {token.address}")
